⏺ Feature flags (флаги функций) — это механизм, позволяющий включать/выключать функциональность в приложении без выпуска нового билда. По сути это конфиг Map<String, bool>, который
  проверяется в коде:

  if (FeatureFlags.isEnabled('voice_messages')) {
    // показать кнопку микрофона
  }

  Зачем это нужно:

  1. Постепенный раскат — включить фичу для 5% пользователей, убедиться что нет крешей, потом раскатать на 100%. Сейчас у вас либо фича есть у всех, либо ни у кого.
  2. A/B тестирование — показать разным группам разный UI и сравнить метрики (retention, engagement).
  3. Kill switch — если новая фича ломает приложение, можно выключить её удалённо за секунды, а не ждать review в App Store (1-3 дня).
  4. Разделение деплоя и релиза — можно мержить недоделанную фичу в master за флагом, не боясь что пользователи её увидят.

  Как реализуют:

  - Firebase Remote Config (бесплатно) — флаги хранятся на сервере Firebase, приложение подтягивает при запуске
  - PostHog / LaunchDarkly / Unleash — специализированные сервисы
  - Простой вариант — эндпоинт на вашем сервере, возвращающий JSON с флагами

  Для MGMess это актуально, но не критично — это корпоративный клиент с ограниченной аудиторией (сотрудники MyGames), где риски раскатки ниже, чем у публичных приложений. Я убрал бы
   этот пункт из приоритетов — Sentry и аналитика принесут больше пользы.
