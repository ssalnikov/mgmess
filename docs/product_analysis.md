# Продуктовый анализ MGMess

> Дата анализа: 2026-03-08
> Версия: Flutter 3.41+ / Dart 3.11+

## Резюме

MGMess — зрелый мобильный клиент Mattermost с 21 экраном, 45+ API endpoints, WebSocket real-time синхронизацией, офлайн-режимом и кастомным расширением MyGames (seens). Из 25 ключевых функций мессенджера **17 реализованы полностью**, **2 частично**, **6 отсутствуют**. Кодовая база чистая (0 TODO/FIXME), архитектура — Clean Architecture + BLoC, 264 теста.

**Главный вывод:** основной функционал мессенджера покрыт, но для полноценной замены десктопного клиента не хватает ряда возможностей, в первую очередь — создания каналов, расширенного поиска, создания групповых DM и link preview.

## 2. Что реализовано частично (требует доработки)

### 2.1 Поиск сообщений — 5/10

**Текущее состояние:** базовый текстовый поиск `POST /teams/{teamId}/posts/search` с debounce.

**Чего не хватает:**
- Фильтры по дате (`before:`, `after:`, `on:`)
- Фильтры по автору (`from:username`)
- Фильтры по каналу (`in:channel-name`)
- Фильтры по типу контента (`has:file`, `has:image`, `has:link`)
- Подсветка найденного текста в результатах
- Поиск внутри конкретного канала (контекстный)
- Недавние поисковые запросы / история
- Chips/подсказки для фильтров

**Влияние:** Поиск — одно из самых частых действий в рабочем мессенджере. Без фильтров пользователи вынуждены скроллить вручную или переключаться на десктоп.

**Сложность:** Medium. API Mattermost уже поддерживает все фильтры через `terms` строку.

---

### 2.2 Создание каналов — 3/10

**Текущее состояние:** можно создать только Direct Message (1-на-1) через кнопку «DM» в профиле пользователя.

**Чего не хватает:**
- Создание публичных каналов (`POST /teams/{teamId}/channels`)
- Создание приватных каналов
- Создание групповых DM (3+ участников, `POST /channels/group`)
- UI: экран создания канала (название, тип, описание, приглашение участников)
- Архивирование каналов
- Удаление каналов (для админов)

**Влияние:** Критическая функция. Пользователи не могут организовать новые обсуждения без доступа к десктопу. Особенно больно для team leads и менеджеров.

**Сложность:** Medium. API готов, нужен только UI.

---

### 2.3 Локализация — 2/10

**Текущее состояние:** UI-строки хардкожены, частично на русском, частично на английском. Пакет `intl` используется только для форматирования дат.

**Чего не хватает:**
- `.arb` файлы переводов
- `AppLocalizations` генерация
- Единообразный язык UI
- Переключение языка в настройках

**Влияние:** Для корпоративного клиента MyGames достаточно одного языка (русский), но текущий mix создаёт ощущение незавершённости.

**Сложность:** Medium. Требуется вынос ~150-200 строк в `.arb` файлы.

## 4. UX-проблемы в существующем функционале

### 4.1 Emoji Picker — 6/10
- Ограниченный набор (~30 стандартных + custom)
- Нет категорий emoji (smileys, animals, food, etc.)
- Нет часто используемых / избранных
- Нет полного Unicode emoji набора
- **Рекомендация:** интеграция пакета `emoji_picker_flutter` или расширение текущего

### 4.2 Настройки уведомлений — глобальные
- Только глобальный фильтр (all / mentions+DM / DM only)
- Нет per-channel настроек (не путать с mute)
- Нет настройки звука/вибрации
- Нет «тихих часов» (DND по расписанию)

### 4.3 Управление каналами
- Нет сортировки каналов внутри категории (ручная, по активности, по алфавиту)
- Нет drag-and-drop между категориями
- Нет пользовательских категорий (создание/переименование)

### 4.4 Чат
- Нет кнопки «scroll to bottom» при чтении истории (есть только при новых сообщениях)
- Нет навигации к конкретной дате
- Нет отметки «(edited)» с tooltip времени редактирования
- Нет preview прикреплённых файлов перед отправкой (миниатюры в поле ввода)

### 4.5 Профиль
- Нет изменения пароля
- Нет информации о последней активности других пользователей («last seen at ...»)
- Нет списка общих каналов с пользователем

## 5. Технический долг и качество

### 5.1 Сильные стороны
- **0 TODO/FIXME** в коде — чистая кодовая база
- **264 теста** (239 unit + 25 integration) — хорошее покрытие
- **Clean Architecture** с чёткими слоями и контрактами
- **Optimistic UI** — реакции, отправка сообщений, mute
- **Exponential backoff** в WebSocket reconnect
- **Retry interceptor** для 5xx ошибок API
- **Auth interceptor** — автоматический logout на 401

### 5.2 Потенциальные улучшения
- Нет crash reporting (Sentry/Crashlytics)
- Нет аналитики использования (Firebase Analytics / PostHog)
- Нет feature flags для постепенного раскатывания
- Нет CI/CD конфигурации в репозитории
- Кеш изображений без лимитов (может расти бесконечно)
- Нет миграций базы данных (при изменении схемы)
