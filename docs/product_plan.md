# Продуктовый план MGMess

## Текущее состояние

**Реализовано:** OAuth GitLab + email авторизация, каналы с поиском и бейджами, чат с пагинацией и WS, треды, черновики, реакции, пиннинг, сохранённые сообщения, упоминания, форвардинг, приоритеты, профили, push (FCM), seens, файлы, typing indicators, свайп для ответа, редактирование сообщений, подтверждение удаления, разделители дат, shimmer-скелетоны, локальная БД (Drift/SQLite) с офлайн-режимом и очередью отправки.

**Критические пробелы:** нет поиска по сообщениям, нет статусов пользователей, нет мьютинга каналов.

---

## Sprint 1-2: Quick Wins (реализовано)

### 1. Свайп для ответа (Swipe-to-Reply)
- **Суть:** Свайп вправо по сообщению мгновенно открывает reply с цитатой — как в Telegram/WhatsApp.
- **Боль:** Сейчас для ответа нужно long-press -> выбрать действие -> начать печатать. Это 3 шага вместо 1.
- **Сложность:** Low.

### 2. Редактирование сообщений (full stack)
- **Суть:** Полноценное inline-редактирование по long-press -> "Edit". API endpoint (`PUT /posts/{id}/patch`) -> DataSource -> Repository -> ChatBloc/ThreadBloc -> UI.
- **Боль:** Опечатки невозможно исправить — пользователи вынуждены удалять и переписывать.
- **Сложность:** Low-Medium.

### 3. Подтверждение удаления сообщений
- **Суть:** Диалог "Вы уверены?" перед удалением.
- **Боль:** Случайное удаление безвозвратно. Раздражает и снижает доверие к продукту.
- **Сложность:** Low.

### 4. Копирование текста сообщения
- **Суть:** Пункт "Copy" в long-press меню. Было реализовано ранее.

### 5. Разделители дат в чате
- **Суть:** Визуальные разделители между днями: "Today", "Yesterday", "Monday, February 22".
- **Боль:** Лента сообщений выглядит как сплошной поток — нет временных ориентиров.
- **Сложность:** Low.

### 6. Скелетоны загрузки (Shimmer)
- **Суть:** Shimmer-скелетоны вместо `CircularProgressIndicator` при загрузке каналов, чата, тредов.
- **Боль:** Спиннер = "приложение тормозит". Скелетон = "контент уже загружается". Воспринимаемая скорость выше на 30%.
- **Сложность:** Low.

## Sprint 3-4: Убираем причины "забить" (реализовано)

### 1. Мьютинг каналов
- **Суть:** Заглушить уведомления для конкретного канала, не выходя из него.
- **Боль:** Шумные каналы (#random, боты) засыпают пушами -> пользователь отключает ВСЕ уведомления -> пропускает важное -> churn.
- **Сложность:** Low. Mattermost API `PUT /channels/{id}/members/{userId}/notify_props` уже поддерживает. Toggle в `MessageActionsSheet` + фильтр в `NotificationBloc`.

### 2. Полнотекстовый поиск сообщений
- **Суть:** Глобальный поиск по всем каналам и внутри конкретного канала. API `POST /teams/{teamId}/posts/search` уже доступен.
- **Боль:** Невозможно найти ссылку/документ/решение, которое кто-то кидал "на прошлой неделе". Пользователи переключаются на десктоп.
- **Метрики:** Feature adoption 40%+ (поиск — один из самых частых действий в рабочих мессенджерах).
- **Сложность:** Medium. UI: search bar + результаты с контекстом + навигация к сообщению.

### 3. Статусы пользователей (Online / Away / DND / Custom)
- **Суть:** Зелёный/жёлтый/красный кружок на аватаре + кастомный статус с эмодзи.
- **Боль:** Непонятно, на месте ли человек. Пользователи пишут "ты тут?" -> лишний шум.
- **Метрики:** Response Rate +20%, количество "мёртвых" сообщений -25%.
- **Сложность:** Medium. WS-события `status_change`, API `/users/status`. Виджет `StatusIndicator`, интеграция в `UserAvatar`, экран установки статуса.

### 4. Pull-to-refresh на экране каналов
- **Суть:** Свайп вниз для обновления списка каналов и непрочитанных.
- **Боль:** Если WS отвалился незаметно — UI "залипает" в старом состоянии.
- **Сложность:** Low. `RefreshIndicator` + `add(RefreshChannels())`.

### 5. Unread badge на иконке приложения
- **Суть:** Число непрочитанных на иконке в launcher (Android) / app icon (iOS).
- **Боль:** Без бейджа пользователь не знает о новых сообщениях, пока не откроет приложение.
- **Сложность:** Low. `flutter_app_badger` + счётчик из `NotificationBloc`.

## Sprint 5-7: Паритет с десктопом (реализовано)

### 1. Локальная БД и офлайн-режим (DONE)
- **Суть:** Кеширование через Drift (SQLite). Отправка из очереди при восстановлении сети.
- **Реализация:** `AppDatabase` с 3 таблицами (Posts, Channels, Users), DAO + Mapper для каждой сущности, `PostLocalDataSource` / `ChannelLocalDataSource` / `UserLocalDataSource`. Репозитории (`PostRepositoryImpl`, `ChannelRepositoryImpl`, `UserRepositoryImpl`) используют `NetworkInfo` для выбора источника данных. `SendQueueService` обрабатывает pending-посты при восстановлении сети.
- **Метрики:** Session Duration +30-40%, DAU retention +15%.

### 2. Медиа-галерея и просмотрщик
- **Суть:** Полноэкранный просмотр изображений/видео со свайпом, зумом, сохранением. Вкладка "Медиа" в info канала.
- **Боль:** Нет навигации между медиа, нет сохранения на устройство.
- **Метрики:** Media sharing rate +35%.
- **Сложность:** Medium. `photo_view` + `PageView` + `GallerySaver`.

### 3. Markdown-тулбар в поле ввода
- **Суть:** Кнопки форматирования (Bold, Italic, Code, List, Link) над клавиатурой.
- **Боль:** Набирать `**жирный**` вручную никто не будет.
- **Метрики:** Formatted messages +50% -> лучше читаемость.
- **Сложность:** Low-Medium.

### 4. Индикатор "Непрочитано" со скроллом
- **Суть:** Автоскролл к первому непрочитанному + плавающая кнопка "N новых сообщений".
- **Боль:** Чат всегда открывается внизу. В активных каналах пользователь теряет контекст.
- **Сложность:** Medium. Сравнение `lastViewedAt` с `createAt`, вычисление offset, управление `ScrollController`.

## My Sprint (реализовано)

### Закреплённые сообщения как "Доска решений"
- **Суть:** Slide-over панель со всеми pinned-сообщениями, группировка по датам. Быстрый доступ из шапки.
- **Боль:** Решения теряются в потоке. Pins мощная фича, но на мобиле нет удобного доступа.
- **Почему "Aha!":** Превращает чат из потока в "базу знаний" команды.
- **Сложность:** Low-Medium. API `GET /channels/{id}/pinned` уже есть.

### Hero-анимации переходов
- Плавные shared element transitions (аватар канала -> шапка чата).
- **Сложность:** Low-Medium. `Hero` widget + `CustomTransitionPage` в GoRouter.

### Haptic feedback на ключевые действия
- Тактильная вибрация при отправке, long-press, pull-to-refresh, реакции.
- **Сложность:** Low. `HapticFeedback.lightImpact()`.

### Навигация из треда в контекст канала
- Из треда — одним тапом к оригинальному сообщению в канале (scroll to message).
- **Сложность:** Medium. `ScrollController.jumpTo()` с вычислением позиции.


## Sprint 8-10: "Aha!"-моменты

### 1. AI-саммаризация чатов
- **Суть:** Кнопка "Краткое содержание" — LLM суммаризирует последние N сообщений: ключевые решения, задачи, вопросы.
- **Боль:** После отпуска/выходных — сотни непрочитанных. Читать всё невозможно.
- **Почему "Aha!":** Ни один корпоративный мессенджер, кроме Slack AI (платного), этого не делает. Для Mattermost — уникальная фича.
- **Сложность:** Medium-High. Бэкенд-сервис (proxy к LLM), UI: bottom sheet с summary.

### 2. Голосовые сообщения
- **Суть:** Кнопка микрофона рядом с Send. Удержание = запись, отпускание = отправка. Свайп влево = отмена.
- **Боль:** Когда руки заняты или проще сказать, чем писать.
- **Почему "Aha!":** Стандарт для Telegram/WhatsApp, но Mattermost mobile этого не умеет.
- **Сложность:** Medium. `record` + `audioplayers`, upload как file attachment.


### 4. FTUE (First Time User Experience)
- **Суть:** При первом входе — быстрый onboarding: "Это ваши каналы", "Свайпните для ответа", "Долгое нажатие для действий". 3 экрана с анимацией.
- **Боль:** Новый пользователь видит список каналов без подсказок. Не понимает, что можно делать. Уходит.
- **Сложность:** Medium.

## Backlog

### Исчезающие сообщения (Disappearing Messages)
- Таймер автоудаления для канала (5 мин / 1 час / 24 часа). Для конфиденциальных обсуждений.
- **Сложность:** High. Требует бэкенд-поддержки (cron job для удаления).

### Быстрые ссылки на Jira/Confluence/GitLab
- Автораспознавание URL задач (PROJ-123), MR, Confluence-страниц -> rich preview.
- **Сложность:** High. Парсинг URL patterns, API-интеграции.
